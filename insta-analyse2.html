This code right here does a few things

  1. it chucks a request at a nicely named url endpoint 'likers', conveniently indicating what i was looking for- people that liked
  2. the code above gathers extremely necessary data related to be able to get the necessary data, gets the 'pk' of each of the posts; although i like to think it means 'pookie' used in 'pookie bear' it more likely means 'primary key', used to identify record in databases. BORING but neccessary.
  3. when looking at it you can see everyone that liked the post, and their usernames
  4. Going through much of the /likers endpoint for different posts there are some similarities of who appears in the data. This means that one person liked multiple posts.
  5. Thus going through the all the /likers endpoint for each of the, and gathering data from the response of each endpoint and storing it in a dictionary format (key:value structure) was a good option.
  6. And thus, the graphing.
